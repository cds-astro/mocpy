# Contributing to MOCPy

MOCPy is based on the IVOA standard for
[Multi-Order Coverage Maps (MOCs)](https://ivoa.net/documents/MOC/20220727/index.html).
Hence, the evolutions of this library should follow the standard. If  your aim is to
change how MOCs work, then the right place to contribute is
https://github.com/ivoa-std/MOC .

## Asking questions, reporting issues

Feedback is also a contribution. We keep an eye on the issues in this repository, or
you can reach the maintainers *via* their email addresses --you can find them in the
`pyproject.toml` file.

When submitting a bug report, please try to write a minimal code example that reproduces
the bug.
